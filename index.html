Resumo

Este projeto consiste no desenvolvimento de um sistema de inventário (CMDB) para uma empresa de grande porte, capaz de armazenar e gerenciar registros de computadores, celulares, servidores, bancos de dados, softwares e sistemas. O sistema utiliza tecnologias front-end para garantir uma interface amigável e intuitiva, enquanto os dados são persistidos localmente no navegador utilizando o localStorage. A abordagem permite o cadastro de itens com campos padrão, como nome, data de compra, modelo/versão e usuário proprietário. Além disso, o projeto visa proporcionar uma solução simples e eficiente para o controle do parque tecnológico da empresa.

Introdução

A gestão adequada dos ativos tecnológicos é fundamental para a operação eficiente de grandes empresas. Com o aumento do número de dispositivos e softwares, torna-se necessário um sistema que facilite o controle e monitoramento desses ativos, prevenindo perdas e otimizando recursos. O CMDB (Configuration Management Database) surge como uma solução para centralizar essas informações, contribuindo para a tomada de decisões estratégicas.

Objetivos

Objetivo Geral

Desenvolver um sistema front-end para gerenciamento de inventário que permita o cadastro e a visualização de ativos tecnológicos.

Objetivos Específicos

  • Implementar formulário de cadastro com campos essenciais (nome, data de compra, modelo, usuário proprietário);
  • Garantir a persistência dos dados utilizando localStorage;
  • Disponibilizar uma página para visualização e edição dos itens cadastrados;
  • Aplicar layout responsivo e profissional utilizando cores turquesa e branco;
  • Fornecer documentação e explicações detalhadas para utilização e testes do sistema.
